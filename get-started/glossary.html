Glossary

Plain-English definitions for every term you'll see in these docs. Use it when a specification feels dense.

Speech fundamentals

ASR (Automatic Speech Recognition)

The process of converting spoken audio into written text. Also called STT (speech-to-text). Shunya's ASR endpoint is POST /v1/audio/transcriptions.

TTS (Text-to-Speech)

The reverse: turning written text into synthesized audio. Also called speech synthesis.

Translation

Converting text (or speech, via ASR + translate) from one language to another. Shunya's translation product is Vāķ.

Voice agent

A conversational system that listens (ASR), understands (NLU / LLM), and speaks (TTS) — usually with a business workflow in between.

Audio concepts

Sample rate

How many audio samples per second. 16 kHz is standard for ASR and TTS output; 8 kHz is standard for telephony (PSTN, Twilio, SIP).

PCM (Pulse Code Modulation)

Raw, uncompressed audio samples. Lowest latency and no decoding overhead — best for real-time pipelines.

μ-law / A-law (G.711)

8-bit, 8 kHz codecs used by telephone networks. μ-law (mulaw) dominates in North America and India; A-law (alaw) in Europe.

VAD (Voice Activity Detection)

Detecting when speech is happening vs silence. Used to chunk audio at natural pauses and to strip silence before sending to the ASR model.

WAV / MP3 / OGG Opus / FLAC

Audio file formats. WAV is uncompressed; MP3 is small and compatible; OGG Opus is great for streaming; FLAC is lossless compressed.

ASR features

Diarization

"Who spoke when" — splitting a transcript into segments by speaker. Each segment gets a speaker: SPEAKER_00 label. Essential for meeting transcripts and call analytics.

Speaker identification

Diarization + named recognition. Registered voice profiles turn SPEAKER_00 into Priya.

Code-switching

Mixing two or more languages in a single utterance ("Can you mujhe that document bhej dena?"). Common across India. Hinglish and Tanglish are code-switch blends. Shunya's zero-codeswitch model handles these natively.

Transliteration vs translation

Transliteration rewrites the same sounds in a different script (Devanagari → Latin: नमस्ते → namaste). Translation changes the meaning into another language (नमस्ते → hello). Shunya exposes both: output_script for transliteration, output_language for translation.

Word timestamps

Per-word start/end times in seconds. Enables karaoke-style subtitle highlighting and precise audio-text alignment.

WER (Word Error Rate)

Transcription accuracy metric. Lower is better. WER = (substitutions + deletions + insertions) / reference_words. Zero STT's composite WER is 3.10%.

TTS concepts

Voice

The speaker identity used for synthesis — vocal timbre, accent, speaking style. Shunya has 46 voices across 23 languages (one male, one female per language).

Cross-lingual synthesis

A voice speaking a language that isn't its native one. Shunya's voices can speak any supported language — voice="Murugan" (Tamil-native) can read English with a Tamil-inflected accent.

Expression style

An emotional tone tag prepended to text: <Happy>, <Sad>, <News>, <Narrative>. Eleven styles available.

Voice cloning

Generating speech in a custom voice from a short reference audio sample (3–6 seconds). Use reference_wav and optionally reference_text.

Prosody

The rhythm, stress, and intonation of speech. Good prosody requires the TTS model to see full sentences — which is why you flush LLM tokens to TTS at sentence boundaries, not word boundaries.

Deployment & security

Sovereign AI

AI systems where compute, data, and model weights stay under the control of the deploying organization — usually within a specific legal jurisdiction. Shunya's on-prem and air-gapped deployments are sovereign by default.

Air-gapped

A deployment with no network connection to the public internet. Used in defence, finance, and healthcare settings where data cannot leave the network.

CPU-compatible / CPU-first

Models designed to run efficiently on standard x86 processors without a GPU. Lowers deployment cost and enables edge / on-device use cases.

SOC 2 Type II

A third-party audit of an organization's security controls over time (usually 6–12 months). Demonstrates operating effectiveness, not just design.

ISO 27001:2022

International standard for information security management. The 2022 revision adds modern controls around cloud services and threat intelligence.

HIPAA

US healthcare privacy law. Covered entities and their vendors must sign a BAA (Business Associate Agreement) before handling protected health information (PHI). Shunya executes BAAs.

FHIR / HL7

Interoperability standards for electronic health records. Shunya's medical STT output can be serialized to FHIR resources or HL7 v2 messages for EHR integration.

Infrastructure

Triton Inference Server

NVIDIA's production inference server. Shunya uses Triton under the hood to serve models at scale. You see it in the /health endpoint response.

RTFx (Real-Time Factor)

How many seconds of audio the model can process per second of wall-clock time. RTFx 100 = 100× real-time. Higher = faster batch throughput.

WebSocket

Persistent bidirectional TCP connection for real-time streaming. Shunya's streaming ASR and TTS both speak WebSocket.

Bearer token

API key passed in the HTTP Authorization: Bearer <key> header.

Intelligence layer

Intent detection

Classifying what a speaker wants (complaint · inquiry · service_request). Pass intent_choices to constrain to your options.

Sentiment analysis

Scoring whether the speech is positive, negative, or neutral. Returns a label, a numeric score, and a one-sentence explanation.

Named Entity Recognition (NER)

Extracting structured data (people, places, dates, amounts) from a transcript.

Keyterm normalization

Standardizing domain-specific vocabulary in a transcript (emi → EMI, nach mandate → NACH mandate).

Profanity hashing

Replacing profane words with **** in the output. Pair with hash_keywords to also mask domain-sensitive terms (card numbers, OTPs).
