Configuration

Every parameter you can pass to POST /v1/audio/transcriptions, in one place. Two are required (file or url, and model); the rest have safe defaults.

Required

FieldTypeDescription
filefile uploadAudio file (WAV, MP3, M4A, OGG, FLAC, WebM). One of file or url required.
urlstringPublic audio URL. Can't be combined with file.
modelstringOne of zero-indic, zero-universal, zero-med, zero-codeswitch.

Language & output

language_code

string · default "auto"

Language hint. ISO 639-1 code (hi, ta, kn, bn, mr, te, gu, pa, ml, or, ur, en) or full name (Hindi, Tamil). Use auto to detect.

Why setting this matters
On clips < 5 seconds, language detection is error-prone and the model can "translate" instead of transcribe. Locking the language avoids both.

response_format

string · default "verbose_json"

Response structure.

ValueOutput
verbose_jsonFull response: segments, NLP, timing, language.
jsonMinimal: {"text": "..."}. OpenAI-compatible.

output_script

string · default "auto"

Transliterate output to a different script without changing the language. Common values: Devanagari, Bengali, Telugu, Tamil, Kannada, Latin, ITRANS. Powered by aksharamukha — no LLM, no latency cost.

# Hindi audio → Devanagari text romanised to Latin
-F "model=zero-indic" -F "language_code=hi" -F "output_script=Latin"
# → "namaste mohammad ji ye ek zaruri call hai"

output_language

string · default none

Translate the full transcript to a target language via Gemini. Result lives in nlp_analysis.translation. Accepts ISO codes or full names. Set this for translated captions; for script change only, use output_script.

Audio pre-processing

use_vad_chunking

bool · default true

Split audio at natural speech pauses using Voice Activity Detection. Silence is stripped before the model sees it. Set false only for continuous dictation with no pauses.

chunk_size

int seconds · default 30

Applies when use_vad_chunking=false. Accepted range 1–60. Sets a fixed segment length.

enable_denoising

bool · default false

Runs noise reduction before transcription. Skipped automatically for files > 5 MB to avoid latency. Use for call-centre and field recordings.

Segmentation & alignment

word_timestamps

bool · default false

Adds per-word start, end, and score to every segment. Uses CTC forced alignment on an ONNX model — runs on GPU, no external call.

enable_diarization

bool · default false

Speaker-level segmentation. Every segment gets a speaker: SPEAKER_XX label; the top-level transcript is prefixed with speaker tags; speakers array lists all unique speakers detected. Segments are capped at 30 s each for quality.

enable_speaker_identification + project

bool · default false

Resolves anonymous SPEAKER_XX labels to registered names (e.g. Priya, Rahul). Requires enable_diarization=true and pre-registered voice profiles. project scopes the speaker library — useful for per-customer isolation.

Speaker registration API →

enable_emotion_diarization

bool · default false

Detects the dominant emotion in each segment and adds an emotion field. Works alongside standard diarization.

Intelligence layer

enable_intent_detection + intent_choices

bool · default false

Classifies the overall transcript intent using Gemini. Optionally constrain to a list of allowed intents by passing intent_choices as a JSON-encoded array. Result appears in nlp_analysis.intent with label, confidence, and reasoning.

-F "enable_intent_detection=true"
-F 'intent_choices=["complaint","inquiry","service_request","compliment"]'

enable_summarization + summary_max_length

bool · default false

Generates a concise transcript summary via Gemini. summary_max_length is an approximate word count (default 150). Result in nlp_analysis.summary.

enable_sentiment_analysis

bool · default false

Returns nlp_analysis.sentiment with a label (positive/negative/neutral), a numeric score, and a short explanation.

enable_keyterm_normalization + keyterm_keywords

bool · default false

Normalises domain-specific terms that the ASR model might render informally. Optionally focus on specific terms by passing keyterm_keywords as a JSON array. Output preserves the original language. Result in nlp_analysis.normalized_text.

Redaction

enable_profanity_hashing

bool · default false

Replaces profane words with **** in-place in both segments[].text and the top-level text. Uses Gemini for detection.

hash_keywords

JSON array · default none

Masks a specific list of words/phrases using regex (case-insensitive, no LLM). Independent of profanity hashing. Use it for account numbers, card numbers, OTP, and any custom sensitive terms.

-F 'hash_keywords=["account number","card number","OTP"]'

Legacy / compatibility

task

string · default "transcribe"

OpenAI-compatible field. Only "transcribe" is supported today.

Putting it all together

curl -X POST https://asr.shunyalabs.ai/v1/audio/transcriptions \
  -H "Authorization: Bearer $SHUNYALABS_API_KEY" \
  -F "file=@call.wav" \
  -F "model=zero-indic" \
  -F "language_code=hi" \
  -F "enable_diarization=true" \
  -F "enable_speaker_identification=true" \
  -F "project=support_team" \
  -F "enable_emotion_diarization=true" \
  -F "word_timestamps=true" \
  -F "enable_intent_detection=true" \
  -F 'intent_choices=["complaint","inquiry","service_request"]' \
  -F "enable_summarization=true" \
  -F "enable_sentiment_analysis=true" \
  -F "enable_denoising=true"
A sane default set
For a contact-centre transcription with agent-assist, start with: model=zero-indic, enable_diarization=true, enable_intent_detection=true, enable_sentiment_analysis=true. Add word_timestamps=true if you need precise search.
