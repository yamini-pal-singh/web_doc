Contact centres — live agent assist + analytics

Real-time transcription, agent-assist suggestions, sentiment tracking, automated QA on every call. Designed to plug into existing CCaaS via SIPREC, WebRTC, or platform-native streaming.

Illustrative blueprint — not a Shunya-published reference architecture
The reference architecture, CCaaS integration patterns, dashboard metrics, and ROI levers below are example designs showing how Shunya's REST + WebSocket APIs can compose with each platform's streaming hooks. They are not officially published Shunya integration guides, and named platforms (Genesys, NICE, Five9, Avaya, Twilio) have not been confirmed as partners on Shunya's public site. Validate with 0@shunyalabs.ai before designing a procurement around these patterns.

Common workflows Illustrative

Live agent assist

  • Call audio forks via SIPREC or CCaaS audio-hook into Shunya's streaming ASR.
  • Partial transcripts appear on the agent desktop in near real time.
  • A sidecar service runs intent + sentiment detection on rolling windows; detected topics trigger suggested replies from a knowledge-base RAG.
  • Escalation rules — e.g. angry sentiment sustained > 30 s — notify supervisors.

Automated QA

  • Every completed call is batch-transcribed with diarization + sentiment + summary.
  • A QA engine scores against a scorecard: mandatory disclosures, greeting compliance, hold time, hang-up behaviour.
  • Scores feed into the analytics warehouse; outlier calls are sampled for human review.

Multi-lingual routing

  • Inbound calls hit a language-detection step. 3–5 seconds of "Hello, how can I help you?" is enough.
  • Calls route to the language-matched agent queue. If queue depth exceeds threshold, fall back to a voice bot that uses Vāķ to translate agent-to-customer in real time.
