Step 1 — Authentication

Create an account, generate an API key, and store it in your environment. Every request carries it as a Bearer token.

Authorization: Bearer sk-your-api-key

See the Quickstart for full setup. Security checklist:

  • Never commit the key to source control
  • Use a secrets manager in production (AWS Secrets Manager, GCP Secret Manager, 1Password, Doppler)
  • Rotate keys if a laptop is lost or a repo is shared externally
  • Create separate keys per environment (dev / staging / prod) so you can revoke one without breaking others

Step 2 — Pick your deployment surface

Cloud API

Fastest to try. Endpoints at asr.shunyalabs.ai and tts.shunyalabs.ai. Pay per minute.

Pick when: prototyping, public SaaS, latency is fine over WAN.

On-prem / VPC

Same containers in your own infrastructure. Data never leaves your network. Custom pricing.

Pick when: regulated industries, data residency, predictable load.

Air-gapped / CPU

Pull open weights from Hugging Face, run on x86. No internet required.

Pick when: defence, offline kiosks, edge devices.

Step 3 — Integrate

Real-time voice bot in 10 lines of code

Below, a minimal loop that streams audio from the browser, transcribes it, sends text to your LLM, synthesizes the reply, and plays it back.

python
import asyncio
from shunyalabs import AsyncShunyaClient
from shunyalabs.tts import TTSConfig

async def voice_bot(audio_stream, llm, client):
    # 1. Stream audio → transcript
    transcript = await client.asr.transcribe(audio_stream, model="zero-indic")
    # 2. Ask the LLM
    reply = await llm.chat(transcript.text)
    # 3. Stream synthesis
    async for chunk in await client.tts.stream(reply,
        config=TTSConfig(model="zero-indic", voice="Sunita", response_format="pcm")):
        yield chunk   # play to speaker as bytes arrive
node
import WebSocket from "ws";

const asr = new WebSocket("wss://asr.shunyalabs.ai/ws");
const tts = new WebSocket("wss://tts.shunyalabs.ai/ws/v1/audio/speech",
  { headers: { Authorization: `Bearer ${process.env.SHUNYALABS_API_KEY}` } });

asr.on("open", () => asr.send(JSON.stringify({
  api_key: process.env.SHUNYALABS_API_KEY,
  model: "zero-indic", language: "hi", sample_rate: 16000, dtype: "int16",
})));
asr.on("message", async (raw) => {
  const msg = JSON.parse(raw.toString());
  if (msg.type === "final_segment") {
    const reply = await llm(msg.text);
    tts.send(JSON.stringify({ model: "zero-indic", input: reply, voice: "Sunita" }));
  }
});
tts.on("message", (chunk) => player.write(chunk));

Optimization knobs worth learning

KnobWhereWhy
enable_denoisingASRCleans noisy call-centre / field audio before transcription.
enable_diarizationASRGets per-speaker labels. Combine with enable_emotion_diarization for emotion.
word_timestampsASRPer-word timing for captions and alignment.
hash_keywordsASRRedact account numbers, OTPs, custom terms — done without an LLM.
response_format=pcmTTSZero decoding overhead for real-time playback.
speed · trim_silenceTTSTighten notifications and IVR prompts.

Step 4 — Handle the hard parts

Interruption handling

In voice agents, the user might talk over the bot. Cancel the in-flight TTS stream when ASR detects speech_start.

See the pattern →
Reconnection & retries

WebSocket drops happen. Implement exponential backoff on ConnectionError, and retry 5xx responses.

See the pattern →
Rate limits

Concurrent request cap is 16 on the default tier. Use an asyncio.Semaphore for batch pipelines; 429s should trigger exponential backoff.

Sentence buffering for LLM streaming

Flush to TTS at . ! ? ; boundaries with a 15-char minimum — never word-by-word. Word chunks kill prosody.

Step 5 — Ship checklist

  • ✅ API key loaded from environment, not code
  • ✅ Errors mapped to user-facing messages (401, 429, 5xx)
  • ✅ Audio deleted from your own storage after transcription
  • ✅ PCM / μ-law for real-time, MP3 for storage
  • ✅ Reconnect logic on WebSocket
  • ✅ Health check (GET /health) wired into your deploy smoke test
  • ✅ X-Request-Id logged — Shunya support uses it for debugging
