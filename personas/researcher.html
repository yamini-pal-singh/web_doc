Step 3 — Run inference locally

Zero STT (Pingala v1 Universal)

from pingala_shunya import PingalaTranscriber

tx = PingalaTranscriber()
segments = tx.transcribe("meeting.wav")
for seg in segments:
    print(f"[{seg.start:.2f} → {seg.end:.2f}] {seg.text}")

Vāķ Translate

import ctranslate2
from transformers import NllbTokenizer

tok = NllbTokenizer.from_pretrained("shunyalabs/vak-translate-1.3b-ct2")
mt = ctranslate2.Translator("shunyalabs/vak-translate-1.3b-ct2", device="cuda")

src = tok("आप कैसे हैं?", return_tensors="np")
tokens = tok.convert_ids_to_tokens(src["input_ids"][0])
result = mt.translate_batch([tokens], target_prefix=[["tam_Taml"]])
print(tok.decode(tok.convert_tokens_to_ids(result[0].hypotheses[0][1:])))
# → நீங்கள் எப்படி இருக்கிறீர்கள்?

Hinglish ASR

from transformers import pipeline

asr = pipeline("automatic-speech-recognition",
               model="shunyalabs/zero-stt-hinglish",
               torch_dtype="float16", device="cuda")
out = asr("call.wav")
print(out["text"])  # mixed Devanagari + Latin tokens preserved

Step 4 — Air-gapped deployment

The typical air-gapped flow:

  1. On an internet-connected machine: huggingface-cli download shunyalabs/pingala-v1-universal --local-dir ./models/pingala
  2. Verify checksums, package the directory as a tarball.
  3. Transfer via signed media into the secure zone.
  4. Load from the local path: PingalaTranscriber(model_path="./models/pingala")
Container parity
If you need production serving in the secure zone, the managed on-prem package includes the same Triton containers used in the cloud. That's the easier path for regulated workloads — contact Shunya Labs ↗.

Step 5 — Fine-tune on your data

Pingala is a transformer encoder-decoder. Standard HF Trainer fine-tuning works. High-level recipe:

  • Collect 10–100 hours of in-domain labelled audio (more if the acoustic conditions are far from pretraining).
  • Format as a Dataset with audio and text columns; 16 kHz mono, 15–30 s clips.
  • Freeze the encoder for the first 1–2 epochs, unfreeze later for final polish.
  • Evaluate on a held-out split using jiwer for WER.
  • Target: <1% WER drop vs pretrained on your domain, or catastrophic forgetting has crept in — lower LR and try again.

Fine-tuning gotchas

  • Language tags: keep the tokenizer's BOS/LID tokens intact — stripping them breaks the multilingual behaviour.
  • Code-switch data: the base Hinglish model is already code-switch-aware. Fine-tuning for a single language often reduces code-switch quality. Keep mixed samples in the training set if you need both.
  • Evaluation: benchmark on OpenASR-style hold-outs (LibriSpeech, AMI, TedLium) in addition to your own data, so you can see whether you're overfitting.
