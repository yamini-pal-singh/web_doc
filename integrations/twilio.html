Twilio

Twilio is the canonical telephony case in Shunya's TTS docs. Both the ASR streaming endpoint and the TTS API support mulaw 8 kHz output — the exact format Twilio Voice expects — so you can wire them together with no resampling step.

Why mulaw matters here

From the TTS docs:

mulaw (G.711 mu-law) and alaw (G.711 A-law) are the two standard codecs for telephone networks globally. Both encode audio at 8 kHz, 8-bit, which is the standard for PSTN, SIP, and most IVR platforms. Use mulaw for Twilio, most Indian PSTN systems, and North American telephony.

TTS for Twilio — IVR prompt example

Synthesise an IVR prompt in mulaw at 8 kHz, ready to send straight to Twilio. trim_silence=True removes dead air at the start and end so the prompt feels tight on the line.

# Full telephony IVR example
config = TTSConfig(
    model="zero-indic",
    voice="Sunita",
    response_format="mulaw",
    trim_silence=True,
)
result = await client.tts.synthesize(
    "<Conversational> नमस्ते! अपना खाता नंबर दबाएँ।",
    config=config,
)
# result.audio_data is ready to send to Twilio / SIP

Streaming TTS into Twilio Media Streams

For voice agents that need audio to start playing before the full sentence is synthesised, open the streaming WebSocket and forward each chunk into Twilio's <Stream> connection. Use response_format="mulaw" on the Shunya stream so chunks arrive in the format Twilio plays back natively.

config = TTSConfig(
    model="zero-indic",
    voice="Varun",
    response_format="mulaw",
)
async for chunk in await client.tts.stream(text, config=config):
    # forward `chunk` to your Twilio Media Streams WebSocket
    await twilio_ws.send_bytes(chunk)

Streaming ASR from a Twilio call

Inbound from Twilio Media Streams arrives as μ-law / 8 kHz audio frames. Shunya's WebSocket ASR accepts that natively — set dtype="ulaw" and sample_rate=8000 in the init message.

// Init JSON sent as the first message to wss://asr.shunyalabs.ai/ws
{
  "api_key": "your-api-key",
  "language": "hi",
  "sample_rate": 8000,
  "dtype": "ulaw",
  "model": "zero-indic"
}

Then forward each binary audio frame from Twilio straight onto the ASR socket — no codec conversion needed.
